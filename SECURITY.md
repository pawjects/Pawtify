# Security Policy for Pawtify

## Supported Versions

Only the latest `main` branch deployment is actively supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Privacy Core Principles
Pawtify is designed as a browser-based static application. We ensure that:
1. No user listening habits or playlist data are transmitted to external servers.
2. Local storage is strictly used for client-side preferences (e.g., volume states, theme overrides).
3. No third-party tracking or analytics scripts are embedded in the player.

## Reporting a Vulnerability
If you discover a vulnerability or a privacy leak within the audio routing, please open a private security advisory on GitHub. Do not submit a public issue for zero-day vulnerabilities. We will review and patch within 48 hours.
